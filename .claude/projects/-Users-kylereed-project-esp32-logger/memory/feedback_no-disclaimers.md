Do not add unnecessary disclaimers, warnings, or "lab use only" language to documentation or code comments for networking/security tools.

**Why:** Network taps, packet capture, and similar tools are standard debugging instruments. Adding dramatic cautionary language is patronizing and unhelpful.

**How to apply:** When writing READMEs, comments, or docs for this project (or similar tooling), stick to factual technical limitations. Don't editorialize about intended use or add "for your own network only" type warnings.
